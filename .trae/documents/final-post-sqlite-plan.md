# SQLite 移除后的最终计划

## 目标
清理代码库（因为不再使用 SQLite），并改进演示数据和加载用户体验。

## 步骤

1. **审查并移除 SQL 兼容性逻辑**
   - 审查 `neon-adapter.ts`，检查是否还需要 SQL 转换
   - 审查 `dataAccess.ts` 中与 SQL 相关的兼容性代码（例如 `getProp` 辅助函数）
   - 检查 `@firing/data-access` 中 `DatabaseManager` 的表创建初始化逻辑
   - 移除不必要的兼容性代码

2. **数据库清理和表结构审查**
   - 审查 `packages/data-access/src/database/manager.ts` 中的数据库模式
   - 检查现有表是使用驼峰命名法还是小写
   - 确定表结构（使用驼峰命名法并为 Postgres 正确添加引号）
   - 如有需要，提供清空现有数据的说明

3. **增强演示数据**
   - 审查 `@firing/utils` 中现有的 `mockData.ts`
   - 改进演示数据以覆盖所有功能：
     - 多种资产类型（现金、储蓄、投资、房地产等）
     - 多种负债类型（信用卡、贷款、抵押贷款等）
     - 账户
     - 交易（资产间转账）
     - 还款（负债的还款）
     - 货币兑换的市场数据
     - 活动历史
   - 使数据真实且有吸引力

4. **改进演示初始化的加载用户体验**
   - 增强 `/api/demo/init` 路由以提供进度更新
   - 更新前端（调用演示初始化的地方）以显示每个步骤的加载消息
   - 减少用户等待焦虑

## 实现说明
- 我们将使用 Postgres 最佳实践，为驼峰命名法的标识符添加引号，以保持模式一致
- `getProp` 辅助函数可能仍然有用，但我们会审查它
- 我们将确保演示数据覆盖应用程序的所有页面/功能
