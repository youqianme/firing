# 添加测试数据功能 - 产品需求文档

## 概述
- **Summary**: 在设置页面中增加“添加测试数据”功能，用于快速生成覆盖所有系统功能的测试数据，以便用户可以直观了解系统的各种功能和展示效果。
- **Purpose**: 解决新用户或开发者在使用系统时缺乏数据导致无法全面体验系统功能的问题，通过一键添加测试数据，快速展示系统的各项功能和数据可视化效果。
- **Target Users**: 新用户、开发者、测试人员

## Goals
- 在设置页面的数据管理部分添加“添加测试数据”按钮
- 实现一键生成覆盖所有系统功能的测试数据
- 测试数据应符合常规业务逻辑，包含各种类型的资产、负债、交易等
- 添加测试数据前进行确认提示，避免误操作
- 确保测试数据能够正确展示在系统的各个模块中

## Non-Goals (Out of Scope)
- 测试数据不需要实时更新或与真实市场数据同步
- 不需要提供测试数据的自定义配置选项
- 不影响系统的现有数据（如果已存在）
- 不需要支持测试数据的增量添加

## Background & Context
- 系统是一个个人资产管理应用，包含多个功能模块：仪表盘、资产、负债、交易、时间轴、收益日历、FIRE 目标等
- 当前用户在首次使用系统时，由于缺乏数据，无法直观了解系统的各项功能和展示效果
- 开发者和测试人员在开发和测试过程中，也需要快速生成测试数据来验证系统功能

## Functional Requirements
- **FR-1**: 在设置页面的数据管理卡片中添加“添加测试数据”按钮
- **FR-2**: 点击按钮后显示确认对话框，提示用户添加测试数据的操作
- **FR-3**: 确认后生成测试数据，包括资产、负债、交易、市场数据、账户、FIRE 配置等
- **FR-4**: 测试数据生成完成后显示成功提示
- **FR-5**: 测试数据应覆盖系统的所有功能模块，确保每个模块都有相应的数据展示

## Non-Functional Requirements
- **NFR-1**: 测试数据生成过程应在 3 秒内完成
- **NFR-2**: 测试数据应符合常规业务逻辑，数据之间应保持一致性
- **NFR-3**: 测试数据的数量应适中，既能覆盖所有功能，又不会导致系统性能问题
- **NFR-4**: 添加测试数据的操作应具有良好的用户反馈，包括加载状态和成功/失败提示

## Constraints
- **Technical**: 测试数据生成应使用现有的数据访问接口，不修改数据库结构
- **Business**: 测试数据应符合常规的个人资产管理场景，避免极端或不合理的数据
- **Dependencies**: 依赖现有的资产、负债、交易等数据模型和 API

## Assumptions
- 系统已存在完整的数据模型和 API 接口
- 用户了解添加测试数据的目的和影响
- 测试数据不会与真实数据混淆（用户会在测试环境或新安装的系统中使用）

## Acceptance Criteria

### AC-1: 界面展示
- **Given**: 用户进入设置页面
- **When**: 查看数据管理卡片
- **Then**: 应看到“添加测试数据”按钮，位于数据管理功能区域
- **Verification**: `human-judgment`

### AC-2: 操作流程
- **Given**: 用户点击“添加测试数据”按钮
- **When**: 系统显示确认对话框
- **Then**: 用户确认后，系统开始生成测试数据，并显示加载状态
- **Verification**: `human-judgment`

### AC-3: 数据生成
- **Given**: 用户确认添加测试数据
- **When**: 系统执行数据生成操作
- **Then**: 系统应生成包含以下内容的测试数据：
  - 多种类型的资产（现金、投资、定期存款等）
  - 不同类型的负债
  - 各类交易记录
  - 市场数据（汇率、黄金价格等）
  - 多个账户
  - FIRE 配置
  - 各种类型的活动记录
- **Verification**: `programmatic`

### AC-4: 功能覆盖
- **Given**: 测试数据生成完成
- **When**: 用户浏览系统的各个功能模块
- **Then**: 每个模块都应显示相应的测试数据，包括：
  - 仪表盘：资产负债概览、趋势图、最近动态
  - 资产：各类资产列表和详情
  - 负债：负债列表和还款计划
  - 交易：交易记录和台账
  - 时间轴：各类活动记录
  - 收益日历：收益热力图和详情
  - FIRE 目标：FIRE 状态和配置
- **Verification**: `human-judgment`

### AC-5: 性能要求
- **Given**: 用户点击确认添加测试数据
- **When**: 系统执行数据生成操作
- **Then**: 数据生成过程应在 3 秒内完成，并显示成功提示
- **Verification**: `programmatic`
