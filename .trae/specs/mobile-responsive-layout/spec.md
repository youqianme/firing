# 移动端响应式布局改进方案

## 为什么 (Why)
目前的 Web 应用在移动设备上的体验较差：
- 数据表格需要水平滚动才能查看完整信息，操作繁琐。
- 固定的底部导航栏会遮挡页面底部的内容。
- 表单输入框在 iOS 设备上可能因字体过小触发自动缩放。
- 表格视图中的操作按钮（编辑/删除）点击区域较小，不易操作。

## 变更内容 (What Changes)
- **布局优化**: 为移动端的主内容区域增加足够的底部内边距 (padding-bottom)，防止内容被底部导航栏遮挡。
- **列表视图优化 (资产、负债、交易)**: 实现移动端专属的“卡片视图”，替代原本的“表格视图”。
  - **桌面端 (Desktop)**: 保持完整的数据表格展示。
  - **移动端 (Mobile)**: 展示垂直排列的卡片列表，每个卡片包含关键信息（名称、金额、图标）及易于点击的操作按钮。
- **表单优化**: 强制移动端输入框字体大小为 `text-base` (16px)，防止 iOS 设备输入时自动缩放。
- **排版调整**: 针对小屏幕调整标题大小和间距。

## 影响范围 (Impact)
- **涉及规范**: 无直接变更，主要是现有功能的体验优化。
- **涉及代码**:
  - `apps/web/app/layout.tsx`: 全局布局调整。
  - `apps/web/app/assets/page.tsx`: 资产列表渲染逻辑。
  - `apps/web/app/liabilities/page.tsx`: 负债列表渲染逻辑。
  - `apps/web/app/transactions/page.tsx`: 交易列表渲染逻辑。
  - 可能涉及全局 CSS 或 Tailwind 配置的微调。

## 新增需求 (ADDED Requirements)
### 需求：移动端卡片视图
系统应在移动设备（视口宽度 < 768px）上以卡片列表形式展示数据。
#### 场景：在手机上查看资产
- **当** 用户在移动设备上访问资产页面时
- **那么** 他们将看到垂直排列的资产卡片列表
- **并且** 每个卡片展示资产名称、图标、金额和账户名称
- **并且** 编辑和删除操作按钮易于点击

### 需求：安全区域支持 (Safe Area Support)
系统应适配现代移动设备（如 iPhone X 及以上）的安全区域。
#### 场景：滚动到底部
- **当** 用户滚动到页面底部时
- **那么** 最后一项内容应完全可见，不被底部导航栏遮挡

## 修改需求 (MODIFIED Requirements)
### 需求：数据表格展示
**旧逻辑**: 数据始终以表格形式展示。
**新逻辑**: 桌面端 (md+) 以表格形式展示，移动端 (base) 以卡片形式展示。
