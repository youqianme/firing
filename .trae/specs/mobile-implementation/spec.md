# 有钱么移动端实现 - 产品需求文档

## 概述
- **摘要**：基于现有Web端实现，完成"有钱么"项目的移动端实现，使用Expo框架开发，实现与Web端一致的核心功能。
- **目的**：为用户提供移动端访问财务数据的能力，实现资产负债管理、交易记录、FIRE目标追踪等核心功能。
- **目标用户**：需要在移动设备上管理个人财务的用户，特别是关注FIRE财务理念的用户。

## 目标
- 实现与Web端一致的核心功能，包括资产负债管理、交易记录、FIRE目标追踪等
- 提供美观、易用的移动端界面，适配不同屏幕尺寸
- 确保数据持久化存储，使用SQLite数据库
- 实现底部标签栏导航，方便用户快速切换功能
- 支持离线使用，数据存储在本地设备

## 非目标（超出范围）
- 不实现与Web端的实时数据同步
- 不支持云存储和多设备数据共享
- 不实现复杂的图表和数据分析功能
- 不支持推送通知和后台同步

## 背景与上下文
- 项目采用Monorepo结构，Web端使用Next.js开发，移动端使用Expo框架
- 数据库结构已在Web端定义，移动端需要保持一致
- 核心功能包括资产负债管理、交易记录、FIRE目标追踪等
- 移动端需要实现与Web端相同的数据模型和业务逻辑

## 功能需求
- **FR-1**：仪表盘功能，显示财务概览（总资产、总负债、净资产、FIRE资产）
- **FR-2**：资产管理功能，支持添加、编辑、删除资产
- **FR-3**：负债管理功能，支持添加、编辑、删除负债，以及记录还款
- **FR-4**：交易记录功能，支持添加、删除交易
- **FR-5**：FIRE配置功能，支持设置年度支出、安全提款率，计算FIRE目标金额
- **FR-6**：基础设置功能，支持设置基础货币、隐私模式
- **FR-7**：数据持久化功能，使用SQLite数据库存储数据
- **FR-8**：导航功能，实现底部标签栏导航

## 非功能需求
- **NFR-1**：性能要求，应用启动时间不超过3秒，页面切换流畅
- **NFR-2**：可用性要求，界面美观、易用，符合移动设备操作习惯
- **NFR-3**：稳定性要求，应用运行稳定，无崩溃现象
- **NFR-4**：兼容性要求，支持iOS和Android主流设备

## 约束
- **技术约束**：使用Expo框架，React Native，SQLite数据库
- **数据约束**：数据模型与Web端保持一致
- **UI约束**：使用React Native内置组件和Expo提供的组件

## 假设
- 用户已了解基本的财务概念和FIRE理念
- 用户使用的是现代移动设备，支持最新的Expo SDK
- 用户接受数据存储在本地设备，不进行云端同步

## 验收标准

### AC-1：仪表盘功能
- **给定**：用户打开应用
- **当**：应用加载完成后
- **然后**：显示财务概览，包括总资产、总负债、净资产、FIRE资产
- **验证**：human-judgment

### AC-2：资产管理功能
- **给定**：用户进入资产管理页面
- **当**：用户点击添加资产按钮
- **然后**：显示资产添加表单，支持填写资产名称、类型、金额等信息
- **验证**：human-judgment

### AC-3：负债管理功能
- **给定**：用户进入负债管理页面
- **当**：用户点击还款按钮
- **然后**：显示还款表单，支持填写还款金额、日期等信息
- **验证**：human-judgment

### AC-4：交易记录功能
- **给定**：用户进入交易记录页面
- **当**：用户点击添加交易按钮
- **然后**：显示交易添加表单，支持选择资产、填写金额等信息
- **验证**：human-judgment

### AC-5：FIRE配置功能
- **给定**：用户进入设置页面
- **当**：用户点击FIRE配置
- **然后**：显示FIRE配置表单，支持设置年度支出、安全提款率
- **验证**：programmatic

### AC-6：基础设置功能
- **给定**：用户进入设置页面
- **当**：用户修改基础货币或隐私模式
- **然后**：设置被保存并生效
- **验证**：programmatic

### AC-7：数据持久化功能
- **给定**：用户添加或修改数据
- **当**：用户退出并重新打开应用
- **然后**：之前的数据仍然存在
- **验证**：programmatic

### AC-8：导航功能
- **给定**：用户在应用中
- **当**：用户点击底部标签栏
- **然后**：切换到对应的页面
- **验证**：human-judgment

## 未解决问题
- 暂无未解决问题，所有功能已在Web端实现，移动端只需保持一致
